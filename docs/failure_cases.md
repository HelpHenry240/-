## 案例 1：S05 误检——厨房台面正常状态被误判为火灾风险

### 错误类型

**误检**（False Alarm）

### 场景描述

S05 是一个正常厨房台面场景。灶台位于台面左侧，上方放着锅；纸巾盒放在台面右侧远端，距离灶台约 60cm。水槽在台面右后方。厨具收纳整齐，纸张与热源保持充分距离，人工标注为无风险。

场景图像：`demo/assets/scenes/S05.svg`

### 人工标注

```json
{
  "ground_truth": []
}
```

人工标注结论：无风险。纸巾盒距离灶台超过 50cm，灶台未开启，不存在火灾隐患。

### 模型输出

```json
{
  "has_risk": true,
  "risks": [
    {
      "type": "火灾风险",
      "objects": ["纸巾", "灶台"],
      "location": "厨房台面",
      "level": "低",
      "reason": "模型把远处纸巾误判为靠近灶台。",
      "suggestion": "保持纸巾远离热源。"
    }
  ],
  "evidence_sufficiency": "",
  "uncertain_points": []
}
```

模型将距离灶台 60cm 的纸巾盒误判为靠近热源，触发了火灾风险规则。

### 可能原因

1. **模型幻觉**：模型看到「纸巾」和「灶台」同时出现在厨房场景中，便自动关联为火灾风险，忽略了二者之间的实际距离。
2. **空间关系错误**：模型对 2D 图像中的距离和位置关系判断不准，无法准确估计纸巾与灶台之间的真实距离。
3. **Prompt 规则不清**：v1 Prompt 中火灾风险规则仅描述为「易燃物靠近热源」，未给出「靠近」的量化标准（如距离小于 30cm），导致模型主观判断过于宽泛。
4. **训练偏差**：模型在训练数据中可能见过大量「厨房纸巾靠近灶台引发火灾」的案例，对这种物体组合产生了偏见。

### 改进方案

1. **细化规则**：在 Prompt 中增加距离阈值。v2 版本已改进：火灾风险要求易燃物与热源距离小于 30cm 才触发，并增加排除条件「热源已关闭且冷却时不触发」。
2. **增加负样本**：在 Prompt 中给出正常场景的负例。v3 版本已改进：负例 2 明确说明「纸巾盒放在远离灶台的台面，距离约 60cm，灶台未开启」应判定为无风险。
3. **要求空间推理**：让模型在 reason 中明确说明距离估计和位置关系，而非仅列出物体名称。
4. **降低温度**：将生成温度从默认值降低至 0.1，减少模型的随机性和幻觉概率。系统已将默认温度设为 0.1。

## 案例 2：S07 等级错误——实验室化学品暴露风险等级被低估

### 错误类型

**等级错误**（Level Error）

### 场景描述

S07 是一个实验室场景。化学品瓶未收纳，放在实验台前侧边缘且标签朝外。实验台旁边有柜子但化学品未放入。化学品处于开放区域，容易被误碰或误用。人工标注为危险物暴露，等级「高」。

场景图像：`demo/assets/scenes/S07.svg`

### 人工标注

```json
{
  "ground_truth": [
    {
      "risk_type": "危险物暴露",
      "objects": ["化学品瓶"],
      "location": "实验台前侧边缘",
      "level": "高",
      "reason": "危险化学品处于开放区域，容易被误碰或误用。",
      "suggestion": "将化学品放入专用柜，并补充标签和安全提示。"
    }
  ]
}
```

### 模型输出

```json
{
  "has_risk": true,
  "risks": [
    {
      "type": "危险物暴露",
      "objects": ["化学品瓶"],
      "location": "实验台",
      "level": "中",
      "reason": "危险品未收纳，处于开放位置。",
      "suggestion": "收纳到专用柜。"
    }
  ],
  "evidence_sufficiency": "",
  "uncertain_points": []
}
```

模型正确检出了危险物暴露风险，但将等级判定为「中」，而人工标注为「高」。

### 可能原因

1. **等级定义模糊**：v1 Prompt 中危险物暴露规则的等级提示仅为「无收纳、无标签或靠近活动区域时为中高风险」，「中高」是一个模糊范围，模型倾向于取中间值「中」。
2. **模型对严重程度判断不一致**：模型可能认为化学品瓶虽未收纳但标签朝外（有一定安全提示），因此判定为「中」而非「高」。但人工标注考虑到化学品处于实验台前侧边缘、容易被误碰的实际情况，判定为「高」。
3. **场景上下文不足**：Prompt 未提供场景类型（实验室）的上下文信息，模型可能不知道这是实验室环境，对化学品的危险程度评估偏低。
4. **位置细节丢失**：模型将位置描述为「实验台」，而人工标注为「实验台前侧边缘」。模型未识别出「前侧边缘」这一关键位置信息，导致对误碰风险的评估不足。

### 改进方案

1. **明确等级标准**：在 Prompt 中给出每个等级的具体定义。v2 版本已改进：危险物暴露规则明确「无收纳且靠近活动区域为高」「有防护但位置偏低为中」「有标签但未收纳为低」。
2. **增加等级判断标准表**：v3 版本进一步用表格固化等级定义，「高=直接危险需立即处理」「中=潜在危险需尽快处理」「低=轻微隐患可择期处理」，减少主观偏差。
3. **增加场景上下文**：在 Prompt 中提供场景类型提示，或通过 `PromptBuilder.build(extra_context)` 注入场景类型信息，让模型结合环境判断危险程度。
4. **要求模型说明等级判断依据**：在 reason 字段中要求模型解释为何选择该等级，便于人工核查和对比。

## 案例 3：S12 漏检——半遮挡电线未被检出

### 错误类型

**漏检**（Miss / False Negative）

### 场景描述

S12 是一个实验室场景。电线横跨实验台前方桌下通道，但部分被桌腿遮挡。从正面单视角观察，电线在图像中占比很小且被桌腿遮挡，不易发现。人工标注为通行/绊倒风险，等级「中」。

场景图像：`demo/assets/scenes/S12.svg`

### 人工标注

```json
{
  "ground_truth": [
    {
      "risk_type": "通行/绊倒风险",
      "objects": ["电线"],
      "location": "实验台前方桌下通道",
      "level": "中",
      "reason": "电线横跨脚部通行区域，且被桌腿遮挡不易发现。",
      "suggestion": "使用线槽固定电线，并清理脚下通道。"
    }
  ]
}
```

### 模型输出

```json
{
  "has_risk": false,
  "risks": [],
  "evidence_sufficiency": "",
  "uncertain_points": []
}
```

模型未检出任何风险，返回 `has_risk: false`，且未在 `uncertain_points` 中报告桌下区域的遮挡情况。

### 可能原因

1. **遮挡**：电线被桌腿遮挡，模型从当前正面视角无法观察到完整的电线路径。这是最直接的原因——目标物体在图像中不可见或部分不可见。
2. **视角不足**：单视角（正面）无法覆盖桌下通道区域。电线位于桌下低处，正面视角的视线被桌面和桌腿阻挡。
3. **小目标**：电线在图像中占比很小，且颜色与地面相近，模型难以从背景中区分出电线。
4. **Prompt 规则不清**：v1 Prompt 中通行/绊倒风险规则虽提到「电线横跨地面通道」，但未强调「桌下、地面低处」等容易遮挡的区域需要特别关注。
5. **模型能力不足**：当前模型对 3D 空间中被遮挡物体的推理能力有限，无法从部分可见的线索推断出被遮挡的电线横跨通道。

### 改进方案

1. **补充视角**：增加侧面或俯视角度的图片。从侧面可以看清桌下通道的电线走向，从俯视角度可以确认电线是否横跨通行区域。这是解决遮挡问题的根本方案。
2. **细化规则**：在 Prompt 中明确要求检查桌下、地面低处等容易遮挡的区域。v2 版本已部分改进：关注区域中增加了「地面、门口、走廊、主要通道（通行宽度是否被压缩）」的描述。
3. **要求证据充分性判断**：在 Prompt 中强调，当视角无法覆盖桌下或地面低处时，必须将 `evidence_sufficiency` 设为「不充分」并在 `uncertain_points` 中报告。三个版本的 Prompt 都包含此约束，但 mock 结果中该字段为空，说明真实模型需要更强的约束或提醒。
4. **更换模型**：尝试能力更强的视觉模型。例如 Qwen-VL-Plus 或 GPT-4.1-mini 对小目标和遮挡场景的处理能力可能优于基础模型。
5. **引入检测 baseline**：先用目标检测模型（如 YOLO、Grounding DINO）定位图像中的电线、桌腿等物体，再将检测结果作为辅助信息提供给 VLM 做关系推理。这种「检测+推理」的两阶段方法可以有效解决小目标和遮挡问题。

## 后续改进优先级

1. **短期（Prompt 层）**：用 v2 或 v3 Prompt 重新跑全量评测，验证误检和等级错误是否改善。
2. **中期（数据层）**：为 S12 增加侧面和俯视视角图片，验证多视角输入是否能解决漏检问题。
3. **长期（工程层）**：引入目标检测 baseline，先定位物体再交给 VLM 做关系推理，系统性地解决小目标和遮挡问题。

每个失败案例都不是孤立的——它们分别代表了巡检系统中三类典型挑战：误检（精度问题）、等级错误（一致性问题）、漏检（召回问题）。通过 Prompt 优化、数据补充和工程手段的递进式改进，可以逐步提升系统的整体可靠性。
